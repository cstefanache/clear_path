The name of the product is "Clear Route"

I want to write an application that is used to perform business optimizations using Genetic Algorithms and have the AI translate the human intent into optimization function (mono or multi-objective) and model the constraints, test known input on the objective function, and be able to run an optimization for a number of iterations to get to the end result. The idea is to give power to non technical executives or VPs to test out optimization problems without having the need to write code. The intent is translated into: Genes, Objective Fuction, Constraints and into the python code that calculates this objectives subject to constraints based on the genes. The underlying framework for running the optimization is PyGan so when translating into it should be in that format. The user should be allowed to test some solutions before runnning the lenghtier process of optimization. The user is allowed to use API keys for various providers: Gemini, OpenAI, Claude or local Ollama by setting up keys and select the model they want to use from the settings panel. 

Functionality:

1. Login / Registration - the user is able to self register an account with the application and then login with the setup credentials
2. The home screen will display the list of Optimization Projects that the user started works on where it can also create a new project
3. Project screen is the screen where the user is creating, running, testing and evaluate optimization results. This is the core functionality of the application and will be detailed below
4. Settings the user can setup API keys for Gemini, OpenAI, Antrhopic models or can setup an IP for a ollama server

The project screen: The user is presented with two side by side panels:
1. Left panel - chat interface - this is the interface where the user can input the information on the problem that he wants to optimize and guides the AI into making changes
2. Right panel:
   1. Definition Tab - this is where the AI is describing the user intent into optimization problem problem parameters: genes (enum, type and limits), more technical description of the user intent into optimization function: purpose and fitness values for each objective function and constraints - the list of constraints that the objective function will be subjected to. All the fields are text fields that the user can edit manually or AI will update based on requests. 
   2. Benchmark Tab - this is a generated UI based on the list of genes defined by the AI (for enums - we have dropdowns) for numeric values we represent input boxes and for string we represent text boxes - the user can click execute button and they will be presented with the values for each optimization function - this panel will be used for the user to test known parameters against the output and validate to check if there is a need to update the objective function before running the optimizaiton process
   3. Executions - this is the panel where one user can perform an execution - they are presented with a way to introduce the number of iterations default to 100, see the progress of the current optimization. For mono-objective functions they can see a graph on how the optimization function minimzed or maximized the value and a table with 5 first solutions and for multi-objective functions they will be able to see the first pareto front as a table. When an execution is completed the AI will perform an interpretation of the output. For mono-objective will interpret the solution found (first) against the objective function and genes; for multi-objective it will represent the boundaries for each objective function against the objective function and genes


Technical details:
 - The application should run in docker with three containers:
    - DB - a postgres database that holds the user information, projects, chats, genes description, objectives description, constraints description, generated python function to calculate the fitness and results
    - Backend - a python fastapi backend
    - Frontend - a javascript material-ui frontend


The genes description output should be in the following format:
- Gene Name: [gene type](decimals & boundaries for numeric, options for string) - gene description and what aspect of the business is capturing
 
The objective description output should be in the following format:
- Problem description - minimization or maximization problem and a brief description of what targets to optimze
- Objectives:
    - Objective function name - description of the objective function and what optimizes in terms of business

The constraints description should be in the following format:
- Constraints:
 - Constraint: description of the constraint and the circumstances it activates

The optimization should be ran with PyGan and the optimization function python code will be generated by the AI dynamically at runtime to be executed and used in the PyGan optimization process.
